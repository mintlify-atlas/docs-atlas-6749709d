OpenAgents Control (OAC) is an AI agent framework built on [OpenCode](https://opencode.ai) that solves a core problem with AI-generated code: it never matches your project's patterns. OAC teaches agents your coding standards once, then generates matching code every time — with mandatory approval gates so you stay in control.

## Why OpenAgents Control?

AI code tools write generic code. You spend hours refactoring it to match your project. OAC fixes this at the source.

<CardGroup cols={3}>
  <Card title="Pattern control" icon="fingerprint">
    Define your patterns once. Agents load them automatically before generating code — no refactoring needed.
  </Card>
  <Card title="Approval gates" icon="hand">
    Agents always propose a plan before executing. Review and approve before any file is written or command is run.
  </Card>
  <Card title="Team-ready" icon="users">
    Commit context files to your repo. Every team member — and every new hire — gets the same standards automatically.
  </Card>
  <Card title="Editable agents" icon="pen-to-square">
    Agents are markdown files you edit directly. No compilation, no vendor lock-in, no black boxes.
  </Card>
  <Card title="Token efficient" icon="bolt">
    MVI principle loads only what's needed. 80% token reduction compared to loading entire codebase context.
  </Card>
  <Card title="Model agnostic" icon="shuffle">
    Use Claude, GPT, Gemini, or local models. Switch providers without changing your workflow.
  </Card>
</CardGroup>

## Get started in 3 steps

<Steps>
  <Step title="Install OAC">
    Run the one-line installer in your project directory to get the developer profile with all core agents and commands.

    ```bash
    curl -fsSL https://raw.githubusercontent.com/darrenhinde/OpenAgentsControl/main/install.sh | bash -s developer
    ```
  </Step>
  <Step title="Add your patterns">
    Run `/add-context` inside OpenCode to answer 6 questions about your tech stack, patterns, and conventions. Takes 10–15 minutes and unlocks the full value of OAC.

    ```bash
    opencode --agent OpenAgent
    > /add-context
    ```
  </Step>
  <Step title="Start building">
    Use OpenAgent for general tasks or OpenCoder for complex features. Agents discover your patterns, propose a plan, and wait for your approval before writing a single line of code.

    ```bash
    opencode --agent OpenCoder
    > "Create a user authentication system with JWT and refresh tokens"
    ```
  </Step>
</Steps>
