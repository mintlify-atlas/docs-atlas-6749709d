An installation profile is a named list of components stored in `registry.json` under the `profiles` key. When you pass a profile name to `install.sh`, the installer fetches `registry.json`, resolves every component in that profile's list, and copies the matching files into your chosen install directory. Profiles exist so you can go from zero to a working agent setup in a single command without manually selecting dozens of individual files.

## Selecting a profile

```bash
./install.sh [profile]
```

Replace `[profile]` with one of the five names below. You can also pipe directly from GitHub:

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/darrenhinde/OpenAgentsControl/main/install.sh) developer
```

## Custom installation

If no profile matches your needs, run the installer without arguments to enter interactive mode:

```bash
./install.sh
```

Interactive mode lets you:

1. Choose an install location (local `.opencode/`, global `~/.config/opencode/`, or a custom path).
2. Select **Custom Install** to pick individual components by type.
3. Review the selection and confirm before any files are written.

You can also list all available components without installing:

```bash
./install.sh --list
```

## Mixing profiles

Install a base profile, then add individual components from another:

<Steps>
  <Step title="Install the base profile">
    ```bash
    ./install.sh developer
    ```
  </Step>
  <Step title="Re-run the installer in custom mode">
    ```bash
    ./install.sh
    ```
    Select **Custom Install**, choose only the additional components you need (for example, `tool:gemini` and `plugin:notify` from the `business` profile), and confirm.
  </Step>
  <Step title="Choose collision handling">
    When prompted, select **Skip existing** to leave your already-installed files untouched and only write the new ones.
  </Step>
</Steps>

## Re-installing and updating

### Add only new components

```bash
./install.sh developer
# When prompted, choose: Option 1 — Skip existing
```

Existing files are preserved; only components not yet present are written.

### Update all components

```bash
./install.sh developer
# When prompted, choose: Option 3 — Backup & overwrite
```

All components are updated to their latest versions. A timestamped backup is created at `.opencode.backup.{timestamp}/` before any files are overwritten.

<Warning>
  **Option 2 — Overwrite all** replaces every file without creating a backup. Any customisations you have made to agent or context files will be lost. Prefer **Backup & overwrite** unless you are certain you want to discard local changes.
</Warning>

### Migrate to a different location

```bash
# Move an existing installation
mv .opencode ~/.config/opencode

# Or perform a fresh install to the new location
./install.sh developer --install-dir ~/.config/opencode
```
